Section 3 — The Agent Landscape
The Common Building Blocks
Agents.md · Skills.md · MCP · Tools — anatomy of every agent framework
Agents.md

The agent's identity, goals, constraints & reasoning style. Not just a prompt — the operating doctrine. Defines who the agent IS and how it thinks under any condition.

Skills.md

Reusable reasoning behaviours: adversarial_reasoning, evasion_selection, report_generation. Think of them as SOPs the agent memorises and applies contextually.

Tools

Individual capabilities exposed via MCP: run_nmap, execute_exploit, send_telegram. Agent selects the right tool from context — no hardcoding required.

MCP — The Bridge

Standardised protocol connecting the agent brain to external tools. The agent calls an MCP endpoint; the server runs the actual tool; structured output returns. One config block unlocks everything.

Section 3 — The Agent Landscape
Why Not Out of the Box?
Four ways a default agent will embarrass you on a live engagement
01 — Context Window Burn

Nmap /24 = thousands of lines. Unmanaged logs fill the context window → reasoning degrades → hallucinations appear. You need surgical ingestion, not a raw dump into the model's working memory.

02 — Hallucination

Wrong CVE IDs, invented tool flags, made-up IP addresses — all executed against live targets without hesitation. Needs constrained tool schemas, output validation, and confidence gating.

03 — No Command Control

Default agents execute whatever the model decides. No scope boundary, no authorisation gating, no escalation limits. This is how you get a very uncomfortable call from a client at 3am.

04 — Integration Gaps

Not wired for kali-mcp, C2 frameworks, or AD enumeration output formats. Requires custom MCP wiring, skill definitions, and output parsers tuned for security tooling response formats.

🤖 A default agent on a live pentest is like giving a new intern root access and saying "just figure it out." Technically possible. Professionally catastrophic.

Section 6 — Implications
What This Means for Defenders
Signature-based detection is increasingly obsolete — detect reasoning, not tools
Legacy Detection — Fragile
  • File hashes (mimikatz.exe) — agent uses in-memory variant
  • Static C2 domains/IPs — agent uses legitimate protocols
  • Specific PowerShell strings — indirect syscalls bypass this
  • Assumes attacker repeats identical behaviour. They won't.
Next-Gen Detection — Resilient
  • Anomalous graph/Cypher queries against AD topology
  • Unusual sequences of tool invocations over time
  • Behavioural chains: recon → cred access → lateral movement
  • Detect reasoning intent — not individual artifact signatures

"Does this sequence of behaviours, taken together, describe an adversarial campaign?"

🛡 If you're still detecting mimikatz.exe by file hash in 2025 — the agent already got Domain Admin and is writing the report.
